## ✍🏻 컨벤션

### 1. Git Flow 전략
- **사용하는 Git Flow 방식**: `main` 브랜치는 배포용, `develop` 브랜치는 개발용으로 사용하며, 기능 개발은 `feature/{기능명}` 브랜치를 생성해 진행합니다.
- 브랜치 네이밍 규칙:
  - `feature/{기능명}`
  - `bugfix/{수정사항}`

### 2. Pull Request 규칙
- **PR 제목 규칙**: `[태그] 작업 내용 요약`
- **PR 작성 시 포함 항목**:
  - 작업한 내용 요약 및 체크리스트 확인
  - 작업 화면 스크린샷 (선택)

### 3. 코드 리뷰 규칙
- **리뷰 시간**: PR 생성 후 24시간 이내에 리뷰.
- **리뷰 기준**:
  - 코드 스타일 준수 여부 (ESLint, Prettier 적용 확인)
  - 기능 동작 확인 및 건설적인 피드백 제공
  - 가독성과 유지보수성 평가
